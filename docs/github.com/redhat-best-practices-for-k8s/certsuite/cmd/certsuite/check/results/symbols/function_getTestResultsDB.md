getTestResultsDB`

| Item | Details |
|------|---------|
| **Package** | `results` (`github.com/redhat-best-practices-for-k8s/certsuite/cmd/certsuite/check/results`) |
| **Signature** | `func getTestResultsDB(filePath string) (map[string]string, error)` |
| **Visibility** | unexported (internal helper) |

### Purpose
`getTestResultsDB` reads a test‑results file and builds an in‑memory lookup table mapping each *test ID* to its result status (`pass`, `fail`, `skip`, or `miss`).  
The function is used by the command that displays check results, enabling quick status retrieval without repeatedly parsing the file.

### Inputs
| Parameter | Type   | Description |
|-----------|--------|-------------|
| `filePath` | `string` | Path to a test‑results file (normally generated by earlier steps of CertSuite). |

### Outputs
- **Map**: `map[string]string` – keys are the test IDs found in the file; values are one of `"pass"`, `"fail"`, `"skip"`, or `"miss"`.  
  The map is populated only for tests that appear in the file; absent tests will not be present.
- **Error**: Non‑nil if any I/O error occurs (e.g., file not found, permission denied) or if the file contains a malformed line that cannot be parsed.

### Key Steps & Dependencies
1. **File Opening**  
   ```go
   f, err := os.Open(filePath)
   ```
   If opening fails, `fmt.Errorf` wraps the error with context.

2. **Scanner Setup**  
   A `bufio.Scanner` scans the file line by line; a large buffer is allocated (`make([]byte, 0, 64*1024)`) to handle long lines safely.

3. **Regex Parsing**  
   ```go
   re := regexp.MustCompile(`(?m)^([^\s]+)\s+(\S+)$`)
   ```
   Each line must match the pattern `<test-id> <status>`. The regex captures two groups: test ID and status string.

4. **Populating Map**  
   For each scanned line:
   - `FindStringSubmatch` retrieves the captured groups.
   - If no match, an error is returned (`fmt.Errorf("invalid line %q", scanner.Text())`).
   - Otherwise, the map entry `[testID] = status` is created.

5. **Resource Cleanup**  
   The file is closed with `defer f.Close()` (or explicitly after scanning). Any error from closing is propagated as well.

6. **Return**  
   On success: `(map[string]string, nil)`.  
   On failure: `(nil, err)`.

### Side Effects
- Reads the entire file into memory line‑by‑line; does not modify the file.
- The function may return an error if the file cannot be opened or contains malformed lines.

### Integration with the Package
`getTestResultsDB` is a low‑level helper called by the higher‑level `checkResultsCmd` command (declared at package level). That command parses CLI flags, loads the results database via this function, and then formats/prints a human‑readable summary. The returned map serves as the core data structure for all subsequent result handling.
