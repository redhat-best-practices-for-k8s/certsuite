getSriovNetworkNodePolicies`

**Purpose**  
Collect all *SriovNetworkNodePolicy* objects that belong to a set of namespaces in a cluster.

The function is part of the automatic discovery logic for the Certsuite operator.  
It is called from higher‑level discovery routines when the operator needs to know
which SR‑IOV network node policies are present so it can, for example, decide
whether an admission webhook should be enabled on those nodes.

---

## Signature

```go
func getSriovNetworkNodePolicies(
    clients *clientsholder.ClientsHolder,
    namespaces []string,
) ([]unstructured.Unstructured, error)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `clients` | `*clientsholder.ClientsHolder` | Holds the Kubernetes REST client used to talk to the API server. The caller typically obtains it from a shared context that already contains all necessary clients. |
| `namespaces` | `[]string` | List of namespace names in which the function should look for SR‑IOV policies. If empty, no namespaces are searched and an empty slice is returned. |

| Return value | Type | Description |
|--------------|------|-------------|
| `[]unstructured.Unstructured` | Slice of unstructured objects representing the found policies. Each element contains the full Kubernetes resource as returned by the API server. |
| `error` | Error object if something goes wrong while listing resources (e.g., permission issues, network failure). A “not‑found” error is treated as a normal empty result rather than an actual error. |

## Key Dependencies

| Dependency | Role |
|------------|------|
| `SriovNetworkNodePolicyGVR` | GroupVersionResource for the custom resource *SriovNetworkNodePolicy*. Used to build the API request path. |
| `clientsholder.ClientsHolder` | Provides access to a dynamic client (`DynamicClient`) that performs the actual API calls. |
| `List`, `Namespace`, `Resource` (from the dynamic client) | Build and execute the list operation against the Kubernetes API. |
| `IsNotFound` (from `k8s.io/apimachinery/pkg/api/errors`) | Detects “resource not found” errors so that they can be treated as empty results. |

---

## How It Works

1. **Early exit** – If no namespaces are supplied, an empty slice is returned immediately.
2. **Iterate over namespaces**  
   For each namespace:
   - Build a list request: `client.DynamicClient.Resource(SriovNetworkNodePolicyGVR).Namespace(ns)`.
   - Execute the request with `.List(ctx, metav1.ListOptions{})`.
3. **Error handling**  
   - If an error occurs that is *not* a “NotFound” error, return it.
   - For `IsNotFound`, ignore and continue (no policies exist in that namespace).
4. **Collect results**  
   Append every item from the returned list (`itemList.Items`) to a cumulative slice.
5. **Return** – After all namespaces are processed, return the collected slice and `nil` error.

---

## Side Effects

* No mutation of global state or passed‑in data; the function is pure in that sense.  
* The only observable effect is the network traffic generated by the API calls.

---

## Relation to the Package

Within the `autodiscover` package this helper centralizes all logic for retrieving
SR‑IOV node policies. Higher‑level discovery functions (e.g., those that decide
whether to enable certain webhook mutators) invoke it to obtain the list of SR‑IOV
policies and then apply further filtering or analysis.

The function is intentionally **unexported** because its behaviour is specific
to the internal discovery flow; callers should use the public API provided by the
package instead.
