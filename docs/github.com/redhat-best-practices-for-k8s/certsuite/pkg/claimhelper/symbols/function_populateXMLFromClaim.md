populateXMLFromClaim`

`populateXMLFromClaim` is a private helper that converts a
`claim.Claim` instance into the XML representation used by the
Junit‑style validation reports generated by **certsuite**.

---

## Purpose

The function builds a `TestSuitesXML` value (the root of the JUnit
XML tree) from:

* the claim data (`claim.Claim`)
* the timestamp when the claim was issued
* the timestamp when the claim was evaluated

It is used only by the package’s public API that writes validation
reports to disk; therefore it has no visible side‑effects outside the
returned value.

---

## Signature

```go
func populateXMLFromClaim(c claim.Claim, issueTime, evalTime time.Time) TestSuitesXML
```

| Parameter | Type          | Description |
|-----------|---------------|-------------|
| `c`       | `claim.Claim` | The claim to serialize. |
| `issueTime` | `time.Time` | When the claim was issued. |
| `evalTime`  | `time.Time` | When the claim was evaluated (validation finished). |

| Return value | Type           | Description |
|--------------|----------------|-------------|
| `TestSuitesXML` | XML root struct | The fully populated JUnit test‑suite tree. |

---

## Key Steps

1. **Create the top‑level suite**  
   *Name*: `"CNF Feature Validation"`.  
   *Timestamp*: current UTC time (`time.Now().UTC()`).

2. **Add a child suite for the claim**  
   The suite name is derived from the claim’s `ClaimID` (e.g.
   `claim-123`).  Its `tests`, `failures`, and `errors` counters are
   set to zero initially.

3. **Translate each validation result**  
   For every entry in `c.Results` a `<testcase>` element is appended to
   the claim suite:
   * **Attributes** – `classname`, `name`, `time` (duration between
     `issueTime` and `evalTime`), `status`.
   * **Status mapping**  
     * `"pass"` → no child elements.  
     * `"fail"` → a `<failure>` element with the failure message.  
     * `"skip"` → a `<skipped>` element.

4. **Calculate suite totals**  
   After all test cases are processed, the claim suite’s `tests`,
   `failures`, and `errors` counts are updated to reflect the results
   collected.

5. **Return the fully built XML structure**

---

## Dependencies

| Dependency | Role |
|-------------|------|
| `time.Now().UTC()` | Provides a timestamp for the root suite. |
| `claim.Claim` fields (`Results`, `ClaimID`) | Source data for test cases. |
| Standard library functions (`append`, `strconv.Itoa`,
  `strings.Split`, `time.Parse`, etc.) | String manipulation, number formatting,
  and time calculations needed to build XML attributes. |

The function does **not** read or write any global state; it is
purely a data‑transforming routine.

---

## How It Fits the Package

* The `claimhelper` package exposes functions that create JUnit
  validation reports from claim data.
* `populateXMLFromClaim` is the core routine called by the public
  writer (`WriteClaimValidationReport`).  
* By keeping this logic isolated, the package can easily unit‑test the
  XML generation without touching file I/O.
