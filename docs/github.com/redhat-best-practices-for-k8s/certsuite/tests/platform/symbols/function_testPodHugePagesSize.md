testPodHugePagesSize`

**Location**

`github.com/redhat-best-practices-for-k8s/certsuite/tests/platform/suite.go:709`

**Purpose**

The function performs a platform‑level validation that the size of huge pages
requested by each pod matches the size that is actually available on the node.
It does this by:

1. Querying all pods that request huge pages (`GetHugepagesPods`).
2. For every such pod, it verifies that the requested huge‑page memory block
   can be satisfied by the node’s configured huge‑page pool using
   `CheckResourceHugePagesSize`.
3. Building a per‑pod report object and storing the outcome in the supplied
   `checksdb.Check` instance.

**Signature**

```go
func testPodHugePagesSize(
    c *checksdb.Check,
    env *provider.TestEnvironment,
    nodeName string,
) func()
```

* `c`: The database entry for this particular check; it is mutated to record the result.
* `env`: The current test environment that holds cluster‑level information (nodes, pods, etc.).
* `nodeName`: Name of the node whose huge‑page configuration should be validated.

The function returns a closure with no parameters and no return value.  
That closure is intended to be executed later by the test harness
(e.g., as part of a `ginkgo.It` block).

**Key Dependencies**

| Dependency | Role |
|------------|------|
| `GetHugepagesPods(env)` | Returns all pods that request huge pages on the cluster. |
| `CheckResourceHugePagesSize(node, pod)` | Validates whether the requested huge‑page size is supported by the node. |
| `NewPodReportObject(pod, ...)` | Builds a structured report entry for a single pod. |
| `SetResult(c, result)` | Persists the final test outcome in the database. |
| `LogInfo`, `LogError` | Emits diagnostic logs. |

**Side‑effects**

* The function writes to the `checksdb.Check` record (`c`) via `SetResult`.
* It emits informational and error messages through the logging helpers.
* No state is mutated on the Kubernetes cluster; it only reads pod & node data.

**How it fits the package**

Within the *platform* test suite, this helper implements a single check that
ensures node‑level huge‑page resources are correctly advertised to pods.  
It is called from higher‑level test definitions (e.g., `TestHugePages`) and
contributes to the overall compliance report generated by CertSuite.
